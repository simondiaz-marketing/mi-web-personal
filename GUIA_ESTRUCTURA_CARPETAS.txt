========================================================================
GUÍA DE ESTRUCTURA Y ARCHIVOS DEL PROYECTO
========================================================================
Este documento ("léeme") explica cómo está organizada la arquitectura 
de este sitio web y para qué sirve cada carpeta y tipo de archivo. 
Es ideal para entender cómo funciona el proyecto "por detrás".

------------------------------------------------------------------------
1. LAS CARPETAS PRINCIPALES
------------------------------------------------------------------------
/src (Source / Código Fuente)
Es la carpeta más importante. Aquí se aloja todo el código de programación 
y diseño del proyecto. Todo el contenido de este directorio es procesado por el 
motor "Astro" antes de publicarse en internet.
  ├── /pages: Cada archivo en este directorio se convierte automáticamente en 
  │           una página web. (Ej: "contacto.astro" crea la URL /contacto)
  ├── /components: Son los "bloques de construcción" reutilizables. Por ejemplo, 
  │                el menú de navegación que aparece en todas las páginas 
  │                se programa una sola vez aquí y se reutiliza a lo largo de la web.
  ├── /layouts: Son las "plantillas" o moldes generales. Contienen la 
  │             estructura base (la cabecera, el pie de página) para evitar 
  │             la repetición de código en cada página individual.
  ├── /styles: Aquí se ubican las hojas de estilo (colores, tamaños, diseño).
  └── /content: Aquí se guardan todos los textos largos y artículos 
                (como las reseñas académicas o narrativas) escritos en Markdown.

/public
Aquí se ubican los elementos que NO requieren procesamiento por parte de Astro. 
Son los archivos "estáticos" (imágenes, íconos, fuentes) que se suben directamente 
al servidor de internet tal cual como están.
  ├── /img: Todas las fotografías, ilustraciones y logos del portafolio.
  └── robots.txt: Un archivo de texto que otorga instrucciones a los 
                  motores de búsqueda (como Google) sobre la indexación del sitio.

/scripts
Contiene pequeños programas automatizados que asisten en el 
mantenimiento técnico, como por ejemplo, un script encargado de convertir 
imágenes pesadas a formatos web optimizados y ligeros.

------------------------------------------------------------------------
2. LOS ARCHIVOS SUELTOS (Configuraciones)
------------------------------------------------------------------------
- package.json: Es el manifiesto del proyecto. Indica qué versión de 
  Astro y qué librerías adicionales (como el generador de sitemap) se necesitan 
  para que el proyecto funcione correctamente.
- package-lock.json: Es el registro exacto y de seguridad que asegura
  que las mismas versiones de las dependencias se instalen en cualquier entorno.
- astro.config.mjs: Es el panel de control del motor Astro. Aquí se 
  activan o configuran características adicionales (como el mapa del sitio).
- vercel.json: Contiene las reglas de seguridad y de enrutamiento que se 
  proporcionan al servidor de Vercel para el despliegue del sitio.
- README.md: El documento de presentación oficial que se muestra al acceder 
  al repositorio principal en GitHub.

------------------------------------------------------------------------
3. DICCIONARIO DE EXTENSIONES (Tipos de Archivos)
------------------------------------------------------------------------
¿Qué significa la extensión al final de cada archivo?

- .astro : Archivos exclusivos del motor Astro. Permiten escribir 
           HTML, CSS y JavaScript de manera combinada y estructurada en 
           un único archivo de fácil lectura.
- .md    : Significa "Markdown". Es un formato de texto enriquecido que
           permite redactar artículos con formato (negritas, enlaces, listas) 
           de forma sencilla sin necesidad de saber programar en HTML.
- .css   : "Hojas de Estilo en Cascada". Son los archivos responsables de
           definir los colores, tipografías, disposición y diseño visual de la web.
- .js    : "JavaScript". Es el lenguaje de programación que otorga 
           interactividad y comportamiento dinámico a la página.
- .mjs   : "Módulos de JavaScript". Es una variante de JavaScript utilizada 
           principalmente para configuraciones internas de servidores o scripts.
- .json  : "JavaScript Object Notation". Son archivos estructurados que utilizan 
           llaves {} para almacenar información y configuraciones en texto 
           simple, legibles tanto por humanos como por sistemas informáticos.
- .webp  : Es un formato de imagen optimizado para la web. Pesa significativamente 
           menos que un archivo .png o .jpg manteniendo una calidad visual similar, 
           lo que asegura tiempos de carga mucho más rápidos.
